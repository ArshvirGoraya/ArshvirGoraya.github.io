Description

A still-in-progress engine and Godot plugin designed to make visual novels either within Godot or entirely separate from it.

Designed by me to make visual novels which could include complex mini-games created within Godot that other visual novel engines cannot allow. Other capabilities of Godot (e.g., particle effects and everything else the engine is capable of) can also be used if this is used as a plugin within the Godot engine.

It is also planned to release as a stand-alone visual novel engine, which would not include all of Godot's capabilities, but offer a simple, code-free way of creating visual novel games.

Program: Godot
Language: GDScript

Capabilities (so far...)

This showcases a inspector plugin I developed for Godot that can take a name and dialogue string and display them within a dialogue box.

This showcases how the plugin allows for sprite changes in the game. It can even allow for viewing which sprite you are selecting by highlighting the options.

  • Each choice leads to a path. When the player goes back and selects a different choice, the prior choice's path gets deleted. If the player wants to re-visit this path, they will have to do everything in that path again instead of just being able to fast-forward through it. In my system, each path is saved and can be re-entered allowing for this fast-forward ability in the path they have already reached.

  • A path is made up of events and multiple changes can occur in a single event. For example, as sprite's frame may change and a corresponding dialogue may appear. Many different changes can occur along side just these two changes as well.

  • Not only can each path be saved and loaded, but the code allows for different saving behaviour.

    A traditional saving method can be toggled where exited paths are deleted. Also, events that happened an arbitrary amount of events ago can be set up to be deleted automatically since the player will likely never go back to these.

Biggest Challenge

Although in a very primitive state, this system will visualize all the paths and events the player has travelled to. It will also allow the player to travel to different events by simply clicking on them.

One of the hardest things was the pathing system. Since I decided that all paths must be re-enterable, creating the system was made much more complicated and went through 3 different iterations before finally being settled on the current one. Most of these changes involved how save data storage was implemented and how the user would experience navigating the paths themselves. It’s possible that the system will go through other changes as development continues.
